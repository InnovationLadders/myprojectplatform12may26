# نظام الرؤية الخاص/العام لمعرض المشاريع

## نظرة عامة

تم تحديث معرض المشاريع ليدعم نظام رؤية متقدم يسمح للمؤسسات التعليمية بالتحكم في عرض مشاريعها للعامة أو إبقائها خاصة.

## التغييرات الرئيسية

### 1. حقل `is_public` الجديد

تم إضافة حقل جديد `is_public` (boolean) لجدول `gallery_projects`:
- **القيمة الافتراضية للمشاريع الجديدة**: `false` (خاص)
- **القيمة للمشاريع الموجودة بعد الترحيل**: `true` (عام) - للتوافق مع السلوك السابق

### 2. صلاحيات العرض

#### للزوار (غير مسجلين الدخول):
- يمكنهم رؤية المشاريع العامة فقط (`is_public = true`)

#### للمؤسسات التعليمية (role: school):
- يمكنهم رؤية جميع مشاريعهم (العامة والخاصة)
- يمكنهم تبديل حالة المشروع بين عام وخاص
- المشاريع الجديدة تكون خاصة بشكل افتراضي

#### للإداريين (role: admin):
- يمكنهم رؤية جميع المشاريع من جميع المؤسسات
- يمكنهم تعديل أي مشروع وتبديل حالة الرؤية

### 3. الواجهة والمميزات الجديدة

#### في صفحة المعرض (Gallery.tsx):

1. **إحصائيات محدثة**:
   - عدد المشاريع العامة
   - عدد المشاريع الخاصة (للمؤسسات والادمن فقط)
   - إجمالي المشاريع

2. **فلتر الرؤية** (للمؤسسات والادمن فقط):
   - الكل
   - عامة فقط
   - خاصة فقط

3. **مؤشرات بصرية على بطاقات المشاريع**:
   - شارة خضراء مع أيقونة كرة أرضية (🌐) للمشاريع العامة
   - شارة برتقالية مع أيقونة قفل (🔒) للمشاريع الخاصة

4. **زر تبديل سريع** (Quick Toggle):
   - زر في بطاقة المشروع لتبديل الحالة بين عام وخاص بنقرة واحدة
   - متاح فقط لصاحب المشروع أو الادمن

#### في نموذج إضافة/تعديل المشروع (AddProjectModal.tsx):

1. **Checkbox "عرض المشروع للعامة"**:
   - مع شرح توضيحي للحالة الحالية
   - أيقونة عين مفتوحة (👁) للعام
   - أيقونة عين مغلقة (🚫👁) للخاص

2. **رسالة توضيحية**:
   - تظهر للمؤسسات التعليمية عند إضافة مشروع جديد
   - توضح أن المشروع سيكون خاصاً بشكل افتراضي

### 4. قواعد الأمان (Firestore Rules)

تم تحديث قواعد Firestore لـ `gallery_projects`:

```javascript
// القراءة
allow read: if resource.data.is_public == true ||
               (request.auth != null && (
                 get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin' ||
                 (get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'school' &&
                  resource.data.school_id == request.auth.uid)
               ));

// الإنشاء
allow create: if request.auth != null && (
  get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin' ||
  get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'school'
);

// التحديث والحذف
allow update, delete: if request.auth != null && (
  get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin' ||
  (get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'school' &&
   resource.data.school_id == request.auth.uid)
);
```

## الترحيل (Migration)

### تشغيل الترحيل

1. **الوصول إلى صفحة الترحيل**:
   - يجب على الادمن الدخول إلى صفحة إدارة الترحيل
   - الملف: `/src/pages/admin/MigrateGalleryProjects.tsx`

2. **تشغيل عملية الترحيل**:
   ```typescript
   import { migrateGalleryProjectsToPublic } from '../utils/migrateGalleryProjectsToPublic';
   await migrateGalleryProjectsToPublic();
   ```

3. **التحقق من النتائج**:
   ```typescript
   import { verifyGalleryProjectsMigration } from '../utils/migrateGalleryProjectsToPublic';
   await verifyGalleryProjectsMigration();
   ```

### ملاحظات مهمة عن الترحيل

- ✅ **آمن**: العملية لا تحذف أي بيانات، فقط تضيف حقل `is_public`
- ✅ **قابل للتكرار**: يمكن تشغيل الترحيل عدة مرات بأمان
- ✅ **ذكي**: يتحقق من وجود الحقل قبل التحديث
- ⚠️ **المشاريع الموجودة**: ستصبح عامة (`is_public = true`) للحفاظ على السلوك السابق
- 🆕 **المشاريع الجديدة**: ستكون خاصة (`is_public = false`) بشكل افتراضي

## أمثلة الاستخدام

### للمؤسسات التعليمية:

1. **إضافة مشروع خاص**:
   - أضف مشروع جديد عبر زر "إضافة مشروع"
   - لا تفعّل خيار "عرض المشروع للعامة"
   - احفظ المشروع - سيكون خاصاً بمؤسستك فقط

2. **تحويل مشروع خاص إلى عام**:
   - اضغط على أيقونة القفل (🔒) في بطاقة المشروع
   - سيتحول المشروع إلى عام مباشرة

3. **تحويل مشروع عام إلى خاص**:
   - اضغط على أيقونة الكرة الأرضية (🌐) في بطاقة المشروع
   - سيتحول المشروع إلى خاص مباشرة

### للإداريين:

1. **عرض جميع المشاريع**:
   - افتح صفحة المعرض
   - سترى جميع المشاريع من جميع المؤسسات

2. **تصفية المشاريع**:
   - استخدم فلتر الرؤية (الكل / عامة / خاصة)
   - استخدم فلتر المؤسسات التعليمية

3. **تشغيل الترحيل**:
   - افتح صفحة `/admin/migrate-gallery-projects`
   - اضغط "تشغيل الترحيل"
   - تحقق من النتائج

## الأسئلة الشائعة

### س: ماذا يحدث للمشاريع الموجودة حالياً؟
**ج:** بعد تشغيل الترحيل، ستصبح جميع المشاريع الموجودة عامة (is_public = true) للحفاظ على السلوك السابق حيث كانت جميع المشاريع ظاهرة للجميع.

### س: هل يمكن للطلاب رؤية المشاريع الخاصة؟
**ج:** لا، المشاريع الخاصة مرئية فقط لحساب المؤسسة التعليمية المالكة والإداريين.

### س: هل يمكن للمؤسسة التعليمية تغيير رؤية المشروع بعد نشره؟
**ج:** نعم، يمكن للمؤسسة التعليمية تبديل حالة المشروع بين عام وخاص في أي وقت.

### س: هل يمكن تشغيل الترحيل أكثر من مرة؟
**ج:** نعم، الترحيل آمن ويمكن تشغيله عدة مرات. سيتحقق من وجود الحقل قبل التحديث.

### س: ماذا لو نسيت تشغيل الترحيل؟
**ج:** المشاريع الموجودة ستعتبر عامة بشكل افتراضي (القيمة الافتراضية في الكود هي `true` للتوافق مع السلوك السابق).
