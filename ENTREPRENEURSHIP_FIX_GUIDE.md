# دليل إصلاح مشاريع ريادة الأعمال

## المشكلة

مدير المؤسسة التعليمية لا يرى مشاريع ريادة الأعمال الخاصة بمؤسسته، بينما مدير النظام يراها جميعاً.

## السبب

عند تقديم مشروع لريادة الأعمال، يتم نسخ `school_id` من المشروع الأصلي. إذا كان `school_id` فارغاً أو غير صحيح في المشروع الأصلي، فإن المشاريع المقدمة لن يتم عرضها لمدير المؤسسة.

## الحل

تم إضافة ميزتين لإصلاح هذه المشكلة:

### 1. سجلات تفصيلية للتشخيص

تم إضافة console logs مفصلة في صفحة `EntrepreneurshipSubmissions` لتشخيص المشكلة:

- عند الدخول إلى صفحة ريادة الأعمال كمدير مؤسسة، افتح Developer Console في المتصفح (F12)
- ابحث عن السجلات التي تبدأ بـ `[Entrepreneurship]`
- ستجد معلومات تفصيلية عن:
  - معرف المستخدم (user.id)
  - دور المستخدم (user.role)
  - عدد المشاريع المسترجعة
  - تفاصيل كل مشروع (school_id, school_name, إلخ)

### 2. أداة إصلاح البيانات

تم إنشاء صفحة جديدة في لوحة تحكم المدير لإصلاح البيانات تلقائياً:

#### الوصول إلى الأداة

1. سجل الدخول كمدير نظام (admin)
2. اذهب إلى لوحة التحكم الرئيسية
3. ستجد بطاقة "إصلاح بيانات ريادة الأعمال" في قسم الإجراءات السريعة
4. أو اذهب مباشرة إلى: `/admin/fix-entrepreneurship-data`

#### خطوات الإصلاح

1. **تشخيص البيانات**
   - انقر على زر "تشخيص البيانات"
   - ستظهر لك النتائج:
     - إجمالي المشاريع المقدمة
     - عدد المشاريع الصحيحة (التي لها school_id)
     - عدد المشاريع التي تحتاج إصلاح (school_id فارغ)
     - عدد المشاريع التي تم إصلاحها

2. **مراجعة التفاصيل**
   - ستظهر جدول بتفاصيل المشاريع التي تحتاج إصلاح
   - كل صف يعرض:
     - عنوان المشروع
     - school_id الحالي (MISSING إذا كان فارغاً)
     - school_id الصحيح (من المشروع الأصلي)
     - الحالة (pending/approved/rejected)
     - هل يمكن إصلاحه تلقائياً؟

3. **إصلاح البيانات**
   - إذا وجدت مشاريع تحتاج إصلاح، انقر على زر "إصلاح البيانات"
   - ستظهر رسالة تأكيد، اضغط "نعم"
   - ستقوم الأداة بـ:
     - جلب school_id الصحيح من المشروع الأصلي
     - جلب اسم المؤسسة من قاعدة البيانات
     - تحديث مستند المشروع في مجموعة entrepreneurship_submissions
     - إضافة حقلين جديدين:
       - `fixed_at`: تاريخ ووقت الإصلاح
       - `fixed_note`: ملاحظة توضح أن الإصلاح تم تلقائياً

4. **التحقق من النتائج**
   - بعد الإصلاح، ستظهر رسالة تؤكد عدد المشاريع التي تم إصلاحها
   - يمكنك إعادة تشغيل التشخيص للتأكد من نجاح الإصلاح
   - اطلب من مدير المؤسسة تحديث الصفحة والتحقق من ظهور المشاريع

## التحقق من الإصلاح

بعد إصلاح البيانات:

1. سجل الدخول كمدير مؤسسة تعليمية
2. اذهب إلى قسم "ريادة الأعمال" من القائمة الجانبية
3. يجب أن تظهر جميع المشاريع الخاصة بالمؤسسة
4. افتح Developer Console وتحقق من السجلات:
   ```
   🔍 [Entrepreneurship] Loading submissions for user: {...}
   🏫 [Entrepreneurship] Fetching submissions for school ID: xxxxx
   ✅ [Entrepreneurship] School fetched submissions: X
   ```

## منع المشكلة في المستقبل

الكود الحالي في `useProjects.ts` يضمن أن كل مشروع جديد يتم إنشاؤه سيكون له `school_id` صحيح:

```typescript
const schoolId = user.role === 'school' ? user.id : user.school_id;

const newProject = await createFirebaseProject({
  ...projectData,
  teacher_id: user.role === 'teacher' ? user.id : null,
  school_id: schoolId,  // يتم تعيين school_id تلقائياً
  progress: 0
});
```

وعند تقديم المشروع لريادة الأعمال، يتم نسخ `school_id` من المشروع:

```typescript
const submissionData = {
  project_id: projectId,
  project_title: projectData.title || '',
  teacher_id: projectData.teacherId || '',
  teacher_name: projectData.teacherName || '',
  school_id: projectData.school_id || '',  // يتم نسخه من المشروع
  school_name: projectData.schoolName || '',
  // ... بقية الحقول
};
```

## المشاريع القديمة

المشاريع التي تم إنشاؤها قبل هذا الإصلاح قد تكون فيها بيانات غير كاملة. استخدم أداة الإصلاح لتحديثها.

## ملاحظات تقنية

### قاعدة البيانات

المجموعات المتأثرة:
- `entrepreneurship_submissions`: المشاريع المقدمة لريادة الأعمال
- `projects`: المشاريع الأصلية
- `users`: معلومات المستخدمين والمؤسسات

### الحقول الهامة

في `entrepreneurship_submissions`:
- `school_id`: معرف المؤسسة (يجب أن يطابق user.id لمدير المؤسسة)
- `school_name`: اسم المؤسسة (للعرض فقط)
- `project_id`: معرف المشروع الأصلي

### استعلام قاعدة البيانات

لمدير المؤسسة:
```javascript
const submissionsQuery = query(
  collection(db, 'entrepreneurship_submissions'),
  where('school_id', '==', user.id),  // user.id لمدير المؤسسة
  orderBy('submitted_at', 'desc')
);
```

## استكشاف الأخطاء

### المشاريع ما زالت لا تظهر بعد الإصلاح

1. تأكد من أن user.id لمدير المؤسسة يطابق school_id في المشاريع
2. تحقق من console logs في متصفح مدير المؤسسة
3. تحقق من أن دور المستخدم هو 'school' وليس 'admin'

### أداة الإصلاح لا تجد مشاريع

1. تأكد من وجود مشاريع في مجموعة `entrepreneurship_submissions`
2. تحقق من Developer Console في المتصفح للأخطاء
3. تأكد من أن Firebase متصل بشكل صحيح

### الأخطاء أثناء الإصلاح

- إذا فشل إصلاح مشروع معين، سيظهر في قائمة الأخطاء
- يمكنك محاولة إصلاحه يدوياً من Firebase Console
- أو الاتصال بالدعم الفني

## الدعم

إذا واجهت أي مشاكل:
1. افتح Developer Console في المتصفح (F12)
2. انسخ جميع السجلات المتعلقة بـ `[Entrepreneurship]`
3. شارك السجلات مع فريق الدعم الفني
